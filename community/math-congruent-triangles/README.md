# 📐 Congruent Triangle Criteria | 全等三角形的判定

**Subject**: Math | **Grade**: 8 | **Type**: New Concept | **Language**: Chinese

## Overview

A complete interactive courseware for Grade 8 students learning the five criteria for proving congruent triangles (SSS, SAS, ASA, AAS, HL) and understanding why SSA fails. Includes Canvas-based geometric construction, SVG knowledge graph, worked examples with inline geometry diagrams, proof fill-in-the-blank exercises, and TTS narration.

## Key Features

- 🎨 **Canvas Interactive Verification**: Five independent canvases (SSS/SAS/ASA/HL/SSA) with sliders — students adjust side lengths and angles to see congruent pairs form in real-time
- 🗺️ **SVG Knowledge Graph**: Full prerequisite → core → follow-up topic map with color-coded nodes and directional arrows
- 📐 **SSA Counterexample Demo**: Circle-intersects-ray visualization proves SSA produces two distinct triangles
- ✏️ **Proof Fill-in-the-Blank**: Drag-and-drop chip-to-blank proof exercise with auto-checking
- 📝 **Worked Examples with SVG Diagrams**: Three examples (SSS/SAS/SAS+parallel lines) each with precise inline SVG geometry figures showing tick marks, angle arcs, and annotations
- 🎯 **Three-Level Exercises**: Level 1 (fill-in proof) → Level 2 (method identification) → Level 3 (independent proof)
- 🔊 **TTS Narration**: 13-segment audio walkthrough with progress bar, auto-scroll, and subtitle overlay
- 📊 **Pre-test & Post-test**: 3 + 5 questions for full learning-loop assessment

## Instructional Design

| Element | Implementation |
|:--------|:--------------|
| **ABT Narrative** | "Triangles have 6 elements (And), but measuring all 6 is tedious (But), only 3 key elements suffice — 5 criteria (Therefore)" |
| **Five-Lens Method** | See (Canvas drag verification) → Break (6-combination analysis) → Transfer (proof writing) |
| **Scaffolding** | Level 1: Fill proof blanks → Level 2: Identify the method → Level 3: Write a full SAS proof |
| **Bloom's Coverage** | Remember (mnemonic rhyme), Understand (SSA counterexample), Apply (fill-in), Analyze (method selection), Evaluate (post-test), Create (independent proof) |
