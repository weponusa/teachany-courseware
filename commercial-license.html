💼 商业授权指南

TeachAny Commercial Licensing Guide

TeachAny 采用「AGPL-3.0 + 商业授权」双许可模式。
个人、学术、非营利教育场景免费使用；商业场景需获得授权。

🟢 个人/学术/非营利 — 免费 💰 商业部署 — 需授权 ™️ 商标受保护

一、我需要商业授权吗？

先看你的场景落在下面哪一格：

✅ 无需授权（AGPL-3.0 免费）

  • 👤 个人学习、老师自己备课
  • 🏫 公立学校、非营利教育机构对学生免费使用
  • 🎓 学术研究、论文、案例研究
  • 🌍 开源 fork 并以 AGPL-3.0 继续开源
  • 💡 公司内部教研组自用（不对外收费、不作为产品卖）

💰 需要商业授权

  • 🏢 集成进付费 SaaS 平台
  • 💵 作为付费课程的交付工具（学生付费才能看）
  • 🏫 K12 / 成人 / 企业培训等收费业务
  • 📱 收费 App 上架（App Store / Play Store）
  • 🤝 打包卖给客户（哪怕你的产品主业不是教育）
  • 🔒 闭源二次开发，不想开源修改

⚠️ 特别提醒：AGPL-3.0 的「网络使用即分发」条款意味着——即使你只是把修改后的 TeachAny 部署在你自己的服务器上供别人访问，你也必须提供源码。 想要避免这个义务，就需要商业授权。

    方式二：邮件 weponusa@gmail.com，主题 [TeachAny Commercial] <公司名>

  2. 信息确认（5 个工作日内回复）

    需要你提供：公司营业执照 / 个人身份、拟使用场景说明、预计用户规模与年营收量级、技术集成范围（是否修改源码、是否闭源发布）

  3. 授权协议签署

    标准合同模板（可在 GitHub 公开查看）、电子签名或邮件确认、付款方式支持对公转账 / PayPal / 加密货币

  4. 获得商业授权凭证

    授权证书（PDF，可公开展示）、专属支持通道（Telegram / Slack / 飞书）、升级通道（新版免费获取）

四、常见问题 FAQ

Q1我是非营利学校但对学生收「书本费」，算商业吗？
不算。AGPL-3.0 允许收取实体媒介 / 分发成本费用。只要软件本身不额外收费，不触发商业条款。
Q2我是独立开发者，做个 App 免费上架但有广告，算商业吗？
算。广告收入是商业行为，建议走「🌱 创业者许可」。
Q3我公司内部用，员工培训免费，算商业吗？
不算商业分发，但 AGPL-3.0 的「网络使用即分发」条款仍适用——如果你修改了 TeachAny 并部署在内网服务器供员工访问，员工有权要求你披露源码。 希望闭源修改，则需要商业授权。
Q4我能不能把 TeachAny 的知识树 JSON / 课件内容拿走用？
不能直接拿走用于商业。代码（HTML/JS/CSS/Python）采用 AGPL-3.0 + Commercial 双许可，但 data/trees/ 下的知识图谱结构、 examples/ 下的课件内容享有独立版权，再分发需单独授权。
Q5我 fork 了改了很多，还叫 TeachAny 吗？
AGPL-3.0 只授权软件使用权，不授权商标使用权。Fork 并修改后的版本 不得继续使用「TeachAny」「教我学」这两个名称，也不得使用项目 Logo。取新名字即可。
Q6如果我违反许可怎么办？
违反 AGPL-3.0 或未获商业授权的商业使用构成侵权。我们保留以下权利：① 发出整改通知；② 启动法律程序（中国 / 美国 / 新加坡三地司法辖区均可）；③ 公开披露侵权名单。
Q7可以先试用再决定购买吗？
可以。允许 30 天商业环境评估期（需事先书面声明 intent-to-license），评估期内免费但必须在到期前签约或停止商业使用。

五、为什么采用双许可模式？

TeachAny 的核心诉求是「让每个孩子都有属于自己的可汗学院」——这需要教育资源普惠化，所以：

  • 🟢 普通老师和学生免费用：AGPL-3.0 保证永远免费、永远开源、永远可 fork
  • 💰 商业公司反哺项目：商业授权收入用于维护核心代码、资助贡献者、保持项目长期活力

这是全球主流开源基础设施的标准玩法（MongoDB、Elastic、Redis、GitLab、Grafana 等都采用类似模式）。
